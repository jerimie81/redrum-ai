# Semantic Versioning Policy

`redrum-ai` strictly adheres to [Semantic Versioning 2.0.0](https://semver.org/).

Given a version number `MAJOR.MINOR.PATCH`, increment the:

1.  **MAJOR** version when you make incompatible API changes or significant UX shifts that require user intervention.
2.  **MINOR** version when you add functionality in a backward-compatible manner (e.g., new tools, new subcommands).
3.  **PATCH** version when you make backward-compatible bug fixes.

Additional labels for pre-release and build metadata are available as extensions to the `MAJOR.MINOR.PATCH` format.
